Google PM open-sources Always On Memory Agent, ditching vector databases for LLM-driven persistent memory

VentureBeat – AI · Topic: AI/Cloud/Quantum

  • What it is: Google's "Always On Memory Agent" is an open-source persistent memory system for AI agents, designed without traditional vector databases, relying instead on LLMs to manage structured memory directly.
  • Who: Shubham Saboo, a Google senior AI product manager, open-sourced the "Always On Memory Agent" on Google Cloud Platform's GitHub, enabling commercial usage.
  • What it does: It allows AI agents to continuously ingest and consolidate diverse information, storing structured memories in SQLite without vector databases, using an LLM for direct memory organization and retrieval.
  • Why it matters: This agent simplifies AI infrastructure by replacing complex vector database stacks with LLM-managed memory, reducing operational costs and signaling a shift in future agent development paradigms.
  • Strategic view: This signals a future where AI agents are integrated as persistent, autonomous software systems within enterprise workflows, necessitating robust governance, auditability, and clear operational frameworks beyond technical capabilities.

New KV cache compaction technique cuts LLM memory 50x without accuracy loss

VentureBeat – AI · Topic: AI/Cloud/Quantum

  • What it is: Attention Matching is a novel technique for compressing the KV cache of Large Language Models, which stores working memory for sequential token generation.
  • Who: Researchers at MIT developed Attention Matching, a novel technique designed to significantly reduce the memory bottleneck of large language models' KV cache.
  • What it does: Attention Matching drastically cuts large language model KV cache memory by up to 50x without accuracy loss, outperforming existing techniques in speed and compression.
  • Why it matters: The KV cache bottleneck severely limits enterprise AI applications handling long documents and multi-session tasks, impacting concurrency and driving up expensive hardware resource costs.
  • Strategic view: This breakthrough enables scalable and cost-effective deployment of ultra-long context LLM applications, unlocking new enterprise use cases and significantly enhancing existing AI capabilities.
